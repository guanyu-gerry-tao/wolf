# TAILOR-02 - Stepwise Brief, Resume, And Cover Letter

## Purpose

Verify that the three step commands can be run independently:
`wolf tailor brief`, `wolf tailor resume`, and `wolf tailor cover`.

## Covers

- `UC-06.1.1`
- `UC-07.1.1`
- `AC-04-1`
- `AC-05-1`

## Execution Mode

`ai-reviewed`

## Cost / Risk

- Cost: medium to high
- Risk: external-api
- Requires: `WOLF_ANTHROPIC_API_KEY` or `WOLF_DEV_ANTHROPIC_API_KEY`

## Workspace

Use `WOLF_DEV_HOME=/tmp/wolf-test/acceptance/<run-id>/workspaces/tailor-TAILOR-02`.

## Setup

Use the same CLI-populated fixture profile, resume content, and add-job setup as
[TAILOR-01](TAILOR-01-full-pipeline.md), but under the TAILOR-02 workspace.

## Steps

Run these commands in order:

```bash
WOLF_DEV_HOME=/tmp/wolf-test/acceptance/<run-id>/workspaces/tailor-TAILOR-02 npm run wolf -- tailor brief --job <jobId>
WOLF_DEV_HOME=/tmp/wolf-test/acceptance/<run-id>/workspaces/tailor-TAILOR-02 npm run wolf -- tailor resume --job <jobId>
WOLF_DEV_HOME=/tmp/wolf-test/acceptance/<run-id>/workspaces/tailor-TAILOR-02 npm run wolf -- tailor cover --job <jobId>
```

## Expected Artifacts

- `src/tailoring-brief.md` after `brief`
- `src/resume.html` and `resume.pdf` after `resume`
- `src/cover_letter.html` and `cover_letter.pdf` after `cover`

## AI Review Rubric

Use [`../../reviewers/tailor-artifact-review.md`](../../reviewers/tailor-artifact-review.md)
as the base rubric. The case-specific checks below are added on top.

### Case-specific checks

- `resume.html` and `cover_letter.html` both follow the existing
  `src/tailoring-brief.md` (same candidate story, same JD themes); the brief is
  not regenerated by `tailor resume` or `tailor cover`.

## Pass Criteria

- All three commands exit `0`.
- Dev banner appears on stderr for every wolf invocation.
- Each command stdout is JSON with the expected output path for that step.
- The expected artifacts exist after each step.
- `resume` and `cover` use the same existing `tailoring-brief.md`.
- Shared reviewer rubric returns `PASS` or `PASS_WITH_MINOR_IMPROVEMENTS` and
  the case-specific check above passes.
- No runtime files are written under `~/wolf`, `~/wolf-dev`, or repo-local
  `data/` (ignore the tracked `data/.gitkeep` placeholder).

## Report Requirements

Include command logs, `jobId`, output paths from each JSON response, artifact
existence checks, brief excerpt, AI review findings, and safety checks.
